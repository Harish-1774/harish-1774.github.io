Pursuit Based Guidance for Multi-Agent Formation Control

A hybrid drone formation control system combining Artificial Potential Field (APF) for formation generation and collision avoidance with Detective Deviated Pursuit (DDP; Segal et al., 2005) for efficient leader-follower tracking — implemented and validated on seven Crazyflie 2.1 drones using Crazyswarm and ROS.

Abstract

This project presents the development and implementation of a multi-agent drone formation control system based on the Crazyflie platform and the Crazyswarm ROS framework. The system facilitates coordinated flight of seven quadrotor drones within a constrained workspace, achieving precise formation control and collision avoidance.

A global planner manages formation dynamics, while collision avoidance is accomplished using Artificial Potential Field (APF) methods. To optimize responsiveness and reduce computational overhead, Detective Deviated Pursuit (DDP) is integrated, enabling drones to infer and track dynamic or virtual targets based on observed relative motion changes.

The effectiveness of this hybrid control architecture is validated through successful formation of the drones into predefined letter shapes (I, T, M) and lateral movement, with consistent results demonstrated in both simulation and real-world flight experiments.

Research Objectives

The primary objective was to develop and validate a multi-agent drone formation control framework that:

  • Enables autonomous UAVs to self-organize from arbitrary initial positions into predefined geometric formations using APF methods.
  • Utilizes DDP for efficient leader-follower-based formation tracking and task execution with minimal computational cost.
  • Demonstrates the hybrid strategy through simulation and physical experiments on the Crazyflie 2.1 nano drone platform.
  • Ensures inter-agent collision avoidance, accurate trajectory tracking, and formation stability in constrained environments.
  • Provides a scalable control architecture extensible to outdoor and obstacle-rich deployments.

Hybrid Control Architecture

Formation control problems can be divided into three tasks: formation generation (guiding agents from random positions into a desired topology), formation maintenance (preserving relative configuration during motion), and formation reconfiguration (adapting to disruptions or new goals).

DDP is a pursuit-based leader-follower method with low per-agent computational complexity, but it assumes an already-formed configuration. APF handles initial formation generation and collision avoidance well, but carries higher computational cost. This project integrates both:

  • APF — used to form letter configurations and return drones to standby positions after lateral maneuvers.
  • DDP — used to translate formed shapes laterally across the workspace while maintaining inter-agent spacing.

Detective Deviated Pursuit (DDP)

DDP allows a follower to maintain a constant distance from a leader while deviating its velocity vector by a fixed angular offset relative to the line of sight (LOS). Unlike interception guidance, DDP emphasizes formation keeping by enforcing a fixed range.

One-follower configuration

For a leader L and follower F, with leader speed VL, follower speed VF, range r, LOS angle λ, leader heading offset θ, and constant deviation angle δ:

  • Detective condition (constant range): ṙ = 0 ⇒ VL cos θ − VF cos δ = 0
  • Deviation condition (fixed LOS offset): λ̇ = (1/r)(VL sin θ − VF sin δ)

Path angles relate as γL = θ + λ and γF = δ + λ. If the leader flies straight (γ̇L = 0), then θ = δ.

Multi-follower extension

For N followers, the framework extends recursively using local interactions between consecutive agents. Setting the range rate to zero for follower N gives the required velocity:

VFN = VFN−1 cos(δN−1 − εN−1) / cos(δN)

This recursive formulation scales to larger formations using only local neighbor information.

MATLAB validation

Simulations were run for one-, two-, and three-follower configurations with leader speed 1 m/s and angular velocities of 3°/s, 0.1 rad/s, and 0.06 rad/s respectively. Inter-agent distances remained consistent throughout, validating DDP stability under varying motion dynamics.

Artificial Potential Field (APF) Avoidance

APF provides a local, reactive policy for each agent to reach its target while avoiding nearby agents. For agent Ai at position xi targeting xtargeti, if neighbor Aj enters collision radius rc, a repulsive term is added.

  • Attraction to target: velocity toward the goal, scaled to vmax, zero within goal tolerance εt.
  • Repulsive avoidance: if ‖rij‖ < rc, add a repulsive velocity opposite the relative position vector.
  • Speed constraint: resultant velocity is clipped to vmax.

Total commanded velocity: vi = vtargeti + Σj≠i vavoidij

Python validation

Six agents were simulated in a 2 × 2 m² workspace with random initial and target positions (vmax = 0.1 m/s, rc = 0.28 m). Agents followed straight paths when isolated and dynamically routed around neighbors when within the collision radius. On the Crazyflie stack the same APF law used dt = 0.05 s, vmax = 0.3 m/s, rc = 0.28 m, and a 0.01 m goal tolerance.

Mission Design

Seven Crazyflie drones sequentially form the letters I, I, T, and M in a horizontal plane, then perform lateral translation for each shape.

  1. Takeoff — all seven drones ascend from ground positions.
  2. Formation I (×2) — three drones form a vertical line; four hover in standby. The formation translates laterally, deforms, and returns. This sequence repeats once.
  3. Formation T — five drones form a T shape, translate laterally, then return via APF.
  4. Formation M — all seven drones form an M, translate laterally, then return.
  5. Landing — all drones land simultaneously.

APF forms each letter configuration and handles return maneuvers; DDP drives lateral translation while preserving formation geometry.

Hardware & Software Setup

Crazyflie 2.1 + Lighthouse positioning

The Crazyflie is an open-source nano quadrotor from Bitcraze AB, widely used in research for its modular deck architecture. This project uses the Lighthouse Deck, which receives sweeping IR laser beams from Lighthouse Base Stations and estimates 3D position via onboard photodiodes. Four base stations were deployed for high-accuracy indoor localization, feeding position data into the formation control algorithm.

Crazyswarm

Crazyswarm provides the software stack for coordinating multiple Crazyflies — built on the Crazyflie firmware with support for Lighthouse (or motion capture) tracking. It handles low-level flight control, state estimation, and synchronized multi-agent command broadcasting. High-level navigation and swarm coordination logic were implemented separately on top of its Python API.

ROS 2 middleware

ROS served as the communication middleware. A FormationControl node subscribes to per-drone PoseStamped topics, computes guidance commands, and broadcasts setpoints via Crazyswarm's goTo interface. A multi-threaded executor runs a background spin thread for continuous pose updates while the main thread executes the mission sequence.

Implementation

Software architecture

  • main() initializes Crazyswarm and a FormationControl ROS node on a multi-threaded executor.
  • Pose subscribers receive real-time position and orientation for each drone; quaternion data is converted to Euler angles for DDP kinematics.
  • takeoff() / land() use the allcfs interface for synchronized swarm commands.
  • Mission flow calls formI/T/M, moveI/T/M, and returnI/T/M sequentially with timed pauses between phases.

Formation generation (APF)

Functions formI, formT, and formM simulate all seven agents in a loop using APF velocities. Key parameters:

  • Time step dt = 0.05 s
  • Maximum speed = 0.3 m/s
  • Collision radius = 0.28 m
  • Goal tolerance = 0.01 m

Computed trajectories are downsampled and sent to each Crazyflie via goTo position setpoints at 1 m altitude.

Lateral motion (DDP)

Functions moveI, moveT, and moveM numerically integrate the DDP kinematic equations using solve_ivp (SciPy). Each formation has dedicated kinematics functions encoding leader-follower geometry, deviation angles δ, and inter-LOS angles ε for all followers.

For I and M formations where the leader is collinear with followers, a positive offset is applied to the leader position along its velocity direction so relative position computations remain well-defined.

Leader speed VL = 0.3 m/s. Integration runs over 5–6 s with 101 evaluation points; each solution step is streamed to drones at 0.06 s intervals via goTo.

Return maneuvers

After lateral motion, formations collapse and drones return to standby corners using the same APF guidance law (returnI/T/M), preparing the swarm for the next letter configuration.

Results

Flight experiments successfully demonstrated:

  • Formation I with 3 active drones and 4 in standby hover.
  • Formation T with 5 active drones and 2 in standby.
  • Formation M using all 7 drones simultaneously.
  • Lateral translation of each formation across the workspace with shape preserved via DDP.
  • Collision-free transitions between formations using APF repulsive fields.

Both MATLAB/Python simulations and real Crazyflie flights validated the hybrid APF + DDP architecture. The approach balances computational efficiency (DDP) with robust formation generation and avoidance (APF).

Conclusion & Future Work

This project demonstrates a real-time multi-agent drone formation control system on the Crazyflie platform using Crazyswarm and ROS. Integrating APF for collision avoidance with DDP for pursuit-based formation tracking achieves reliable coordination among seven quadrotors in a constrained indoor environment.

Planned extensions include:

  • Leader-independent DDP — decentralized virtual-leader variants for resilience to single-point failures.
  • Obstacle-rich environments — real-time obstacle detection beyond the open-workspace assumption.
  • Outdoor GPS deployment — for search-and-rescue, agriculture, and infrastructure inspection.
  • Heterogeneous agent coordination — mixed platforms with different sensing and actuation capabilities.
  • Energy-aware path planning — incorporating power constraints into formation and movement decisions.
